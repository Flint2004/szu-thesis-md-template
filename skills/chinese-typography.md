# 中文排版规范

本项目面向中文毕业论文，以下规范是**必须遵守**的，否则构建可能失败或最终稿排版不合格。

## 引号

### 规则

| 场景 | 直引号 `"` `'` | 中文引号 `"` `'` `'` `'` |
|------|:--:|:--:|
| 中文正文段落 | ❌ 禁止 | ✅ 必须 |
| 英文句子 | ✅ 允许 | ❌ 不要 |
| 行内代码 `` `...` `` | ✅ 保留 | ❌ 禁止 |
| 围栏代码块 ```` ```...``` ```` | ✅ 保留 | ❌ |
| 数学公式 `$...$` | ✅ 保留（含导数撇号 `x'`） | ❌ |
| URL / 路径 | ✅ 保留 | ❌ |
| pandoc 语法 `::: {custom-style="xxx"}` | ✅ 必需 | ❌ 会破坏 pandoc 解析 |
| 参考文献条目 | ✅ 保留（英文期刊名有时含撇号） | ❌ 不动 |

### 写法来源

- 搜狗/微软拼音等中文输入法在中文模式下**默认输出** `"`/`'`（U+201C / U+201D / U+2018 / U+2019）
- 从 OpenAI / GitHub / 论文 PDF 拷贝英文 → 中文，容易夹带直引号，需检查
- 从旧 docx 粘贴的文本，pandoc 会自动识别；不需额外处理

### 批量修正

若发现直引号混入中文段落（视觉上不容易辨认），可用 Python 脚本批处理。必须保护代码块/公式/URL/参考文献，简单的 `sed 's/"/"/g'` 会误伤。

推荐方式：派生 Agent 执行，给出的 prompt 模板见 `writing-full-thesis.md` 末尾自检部分；或参考本仓库历史 commit `e.g. "replace direct quotes"`。

## 空格

- **中英文混排空格**：中文与英文/数字之间**不加**空格（pandoc 的 CJK 渲染会自动处理）
  - 例外：专有名词/缩写作为独立词，建议加，如 "使用 Softmax 输出层"
- **全角空格 U+3000**：用于目录大标题 "目　录"、题名首字缩进等
- **标点后空格**：中文标点（`，` `。` `；`）后**不加**空格；英文标点（`,` `.` `;`）后**加**半角空格

## 标点

- 列举用 `、`（顿号），不用 `,`
- 括号：中文段落内嵌英文短语时用**全角括号** `（...）`；括号内纯英文/公式时可用半角 `(...)`
- 破折号 `——`（两个中文长横）
- 省略号 `……`（两个中文，6 个点）
- 连字符：中文混排仍用半角 `-`（如 "CIFAR-10"、"one-vs-rest"）
- 数字范围用半角 `-`（如 "10-20 轮"）

## 数学

- 行内：`$...$`
- 独占行：`$$...$$`
- **不要**用 `\(...\)` 或 `\[...\]`（pandoc 默认关闭）
- 变量斜体、常数/函数名直立：函数名用 `\operatorname{Norm}` 或 `\text{sign}`
- 数字自动直立：**脚本后处理**把所有 OMML 纯数字 run 加 `<m:sty val="p"/>`；markdown 源不需干预

## 文件编码

- 全部 UTF-8（无 BOM）
- 换行符 LF（Unix），避免 CRLF

## 常见检查命令

```bash
# 扫描直引号（排除代码块和公式比较粗糙，人工核对）
grep -nP '[a-zA-Z0-9\u4e00-\u9fff]\"[\u4e00-\u9fff]' thesis/*.md

# 扫描疑似误用的英文逗号/句号在中文后
grep -nP '[\u4e00-\u9fff][,.]\s' thesis/*.md

# 扫描全角空格用在不合适的位置（除目录标题外）
grep -n $'\u3000' thesis/*.md | grep -v '目　'
```

## 常见错误示例

```diff
- 我们提出了"可微 SVM"框架。
+ 我们提出了"可微 SVM"框架。

- 见 Section 3.2, 该节详述了...
+ 见第 3.2 节，该节详述了……

- 性能对比(表 5.1)显示:
+ 性能对比（表 5.1）显示：

- 数据集包括CIFAR-10和CIFAR-100。
+ 数据集包括 CIFAR-10 和 CIFAR-100。
```
