# 写完整一篇论文

把多章内容一次性产出的流程。适用于：在占位模板上替换为真实毕设内容。

## 前置

- 读完：`writing-chapter.md` / `figures-and-tables.md` / `citations-and-references.md` / `chinese-typography.md`
- 确认主题、研究问题、贡献点、章节大纲已与导师/用户对齐
- 深大本科毕设硬性要求：
  - 正文 ≥ 12000 中文字
  - 参考文献 ≥ 10 篇、外文 ≥ 2 篇
  - 中文摘要 ≥ 200 字、关键词 3-4 个、每词 ≤ 6 字
  - 英文摘要由中文翻译而来

## 通用结构（建议保留）

```
00_frontmatter.md  目录占位（# 【摘要】隐藏）+ 中文题名 div + 中文摘要 + 关键词
01_intro.md        # 1 绪论
                     ## 1.1 研究背景与意义
                     ## 1.2 国内外研究现状
                     ## 1.3 研究内容与主要贡献
                     ## 1.4 论文组织结构
02_related.md      # 2 相关工作与理论基础
03_method.md       # 3 方法章（建议：符号→模型→子模块→算法流程）
04_theory.md       # 4 理论分析章（若选题偏理论）
05_experiments.md  # 5 实验与结果分析
06_conclusion.md   # 6 总结与展望
07_references.md   # 参考文献（≥10 篇、外文≥2）
08_ack.md          # 致谢
09_abstract_en.md  # 【Abstract】隐藏 + 英文摘要 + 【Key words】
10_appendix.md     # 附录 A 证明 / 数据 / 补充材料
```

## 写作顺序建议

1. **先骨架**：每章写出 H1/H2/H3 标题 + 每节一两句说明，确认整体结构合理
2. **再摘要**：基于骨架写摘要（中文先行，英文后翻）
3. **后填肉**：按章节填充，边写边补充参考文献条目到 `07_references.md`
4. **最后校对**：引用编号、图表编号、标点统一

## 章节间风格协调

- 同一概念在全文使用同一译名（首次出现时给中文+英文，如"支持向量机（Support Vector Machine，SVM）"）
- 图表编号按章，`图 N.M` / `表 N.M`
- 不要在多章反复重复同一段背景介绍；交叉引用用"详见第 N 章"
- 避免第一人称；用"本文"、"本章"、"本节"等中性措辞

## 敏感措辞（避免）

这是本科毕设而非会议论文，**不要**出现：
- "针对审稿人的关注…"、"响应 reviewer 建议…"
- "改编自某会议论文"、"根据 rebuttal 数据…"
- 任何暗示投稿/rebuttal 上下文的句子

如遇类似表述，改写为中性陈述（"为评估…，本节在…上测量了…"）。

## 参考文献

- 按正文首次出现顺序编号
- 格式遵循 GB/T 7714-2015（示例见 citations-and-references.md）
- 英文作者姓全名、名缩写；作者 > 3 加"等" / "et al."
- **假如涉及模型/虚构论文**（如演示模板），参考文献也要编得像真的（人名、期刊卷期页码齐全）

## 写完后自检清单

- [ ] 中文字数 ≥ 12000（用 `wc -m thesis/0[1-6]*.md` 粗估）
- [ ] 参考文献 ≥ 10 篇、外文 ≥ 2 篇
- [ ] 摘要 ≥ 200 字
- [ ] 所有图/表都有 caption 且被正文引用
- [ ] 所有 `[N]` 引用都能在参考文献找到对应条目（`grep -oE '\[\d+(?:[-, \d]+)?\]' thesis/*.md` 对照 `grep '^\[' thesis/07_references.md`）
- [ ] 中文引号全部是 `"`/`'` 而非 `"`/`'`（见 chinese-typography.md）
- [ ] 章节编号连续、无跳跃
- [ ] 图表序号连续、与引用匹配

## 构建

```bash
uv run python scripts/build.py
# → build/thesis.docx
```

打开后 Ctrl+A F9 更新所有字段（目录、页码、交叉引用）。
