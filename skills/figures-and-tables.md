# 图片与表格

## 图片

### 插入方式

```markdown
![图 N.M 图片说明文字。可以写得很长，pandoc 会作为 caption 渲染在图下方，脚本自动居中并设为 9pt 小五。](../figures/filename.png)
```

- **caption 必须以 "图 N.M "** 开头（中间半角空格）。脚本按此识别并套 caption 样式。
- 路径相对 `thesis/` 目录：`../figures/xxx.png`。
- 推荐 PNG（线条清晰）或 JPG（照片）。
- 图片宽度会被自动限制到 ≤ 9.14 cm（正文页宽的 60%），cy 等比缩放。

### 对齐与缩放

- 所有图片默认**居中**（段落属性 + drawing 段落居中）
- 无需在 markdown 中指定宽度；如需局部例外，可用 pandoc 的 `![caption](path){width=60%}` 显式覆盖

### 引用图片

在正文中引用图：

```markdown
如图 3.2 所示，参数 $\theta$ 在 0.5 附近出现最大值。
```

这种**引用句**不会被误判为 caption：脚本通过 "pStyle=ImageCaption" 或"紧邻表格"双条件识别真 caption，普通正文里以"图 N.M"开头的句子不受影响。

## 表格

### 插入方式

```markdown
**表 N.M** 表标题，「表 N.M」用 `**...**` 加粗。下一行紧接 markdown 表格。

| 列名 1 | 列名 2 | 列名 3 |
|:-------|:------:|-------:|
| 单元格 | 单元格 | 单元格 |
| 单元格 | 单元格 | 单元格 |

表格结束后**空一行**再继续正文。
```

### 对齐符号

- `:---` 左对齐
- `:---:` 居中
- `---:` 右对齐
- `---` 默认（一般左）

### 脚本自动处理

脚本识别"段落紧邻其后是 `<w:tbl>`"的段为表 caption，套以：

- 居中 + 9pt 小五
- "表 N.M" 加粗（因 markdown 写了 `**...**`，pandoc 自动保留）

对表格本身套以：

- 整表居中（`tblPr/jc=center`）
- booktabs 三线：顶线粗（1.5pt），表头底线细（0.5pt），表底线粗（1.5pt），无竖线无中间横线
- 表头行加粗、单元格垂直居中
- 每个单元格内文本居中、无缩进

### 常见坑

- ❌ caption 和表格之间空行再多个段落 → 识别失败
- ✅ caption 紧接表格（空一行即可）
- ❌ 表格内部放 markdown 代码块 / 含管道 `|` 的字符 → 破坏表格语法；用 `\|` 转义
- ❌ 多页长表 → pandoc 不支持自动分页表头重复；超长表拆成多张

## 图表交叉引用（简化方案）

目前**不使用** pandoc crossref。正文中直接写"图 3.2"、"表 5.1" 即可（纯文本）。
如需真正的可点击交叉引用，自行加 `{#fig:label}` 标签 + `@fig:label` 引用并启用 `pandoc-crossref` 过滤器，超出本模板基础能力。

## 图文件管理

- 放 `figures/`，文件名用小写 + 下划线：`learning_curve.png`、`feature_tsne.png`
- 大文件（> 5MB）建议用外部图床或 Git LFS；仓库 `figures/` 已纳入 git 追踪
- 截图/示意图建议附原 svg / drawio 源文件（方便后续修改）

## 自检

```bash
# 列出所有图片引用
grep -E '^\!\[' thesis/0[1-6]*.md

# 列出所有表 caption
grep -E '^\*\*表 [0-9]+\.[0-9]+\*\*' thesis/0[1-6]*.md
```

对照正文"如图/表 X 所示"的引用数目，两者应匹配。
