# 引用与参考文献

## 正文引用

裸写方括号加数字，脚本自动转**上标超链接**（点击可跳转）：

| 源写法 | 渲染效果 | 说明 |
|--------|---------|------|
| `[1]` | ¹（超链接） | 单引用 |
| `[2-4]` | ²⁻⁴ | 范围引用，链接指向起点 `ref_2` |
| `[3, 7]` | ³,⁷（两个独立超链接） | 多引用 |
| `[0, 1]` | `[0, 1]`（原样保留） | 以 0 开头不改；供数学区间 `$[0,1]$` 等用 |

用法示例：

```markdown
相关工作见 [1]；经典理论由 [3-5] 建立；近期进展综述可参阅 [6, 8, 12]。
```

## 参考文献条目（`thesis/07_references.md`）

**格式**：GB/T 7714-2015 信息与文献 参考文献著录规则。每条顶行写起，脚本自动套悬挂缩进 + 小五字号。

**顺序**：按正文中首次出现的顺序编号，从 1 开始。

### 条目模板（按类型）

```markdown
[N] 作者. 文献题名[J]. 刊名, 出版年, 卷号(期号): 起止页码.
```

类型标识：

- `[J]` 期刊
- `[M]` 专著
- `[C]` 会议论文集
- `[D]` 学位论文
- `[R]` 报告
- `[N]` 报纸文章
- `[P]` 专利
- `[S]` 标准
- `[EB/OL]` 电子文献/在线资源

### 示例

```markdown
[1] LECUN Y, BENGIO Y, HINTON G. Deep learning[J]. Nature, 2015, 521(7553): 436-444.

[2] HE K, ZHANG X, REN S, et al. Deep residual learning for image recognition[C]//Proceedings of the IEEE Conference on Computer Vision and Pattern Recognition. 2016: 770-778.

[3] 袁庆龙, 候文义. Ni-P 合金镀层组织形貌及显微硬度研究[J]. 太原理工大学学报, 2001, 32(1): 51-53.

[4] 刘国钧, 郑如斯. 中国书的故事[M]. 北京: 中国青年出版社, 1979: 115.

[5] 张凯军. 轨道火车及高速轨道火车紧急安全制动辅助装置: 201220158825.2[P]. 2012-04-05.

[6] 中国互联网络信息中心. 第 29 次中国互联网络发展现状统计报告[R/OL]. (2012-01-16)[2013-03-26]. http://www.cnnic.net.cn/hlwx/2012/P0201452640.pdf.
```

### 规则要点

- 欧美作者：**姓全大写**，名缩写为首字母大写
- 作者 > 3 人：列前三名后加 `等` 或 `et al.`
- 所有符号用**半角**（包括括号、逗号、冒号）
- 出版年与卷期之间逗号，卷号后括号接期号，冒号后页码
- 卷期页码之间不能漏空格或标点

## 数量要求（深大本科毕设）

- 参考文献总数 ≥ 10 篇
- 外文文献 ≥ 2 篇
- 艺术/建筑类等专业可 ≥ 8 篇，外文 ≥ 2 篇不变

## 自检脚本

```bash
# 列出正文所有引用编号
grep -ohE '\[[0-9]+(-[0-9]+)?(, *[0-9]+(-[0-9]+)?)*\]' thesis/0[0-689]*.md | sort -u

# 列出所有参考文献条目首数字
grep -oE '^\[[0-9]+\]' thesis/07_references.md | sort -u

# 两者应一一对应；漏缺或多余会被脚本后处理识别但不报错
```

## 注意事项

- **不要**在参考文献条目中使用 Markdown 特殊字符（`*` `_` `[` `]` 的非配对使用会被 pandoc 误解析；如需包含，可用反斜杠转义）
- **不要**把 URL 写在 `<>` 里，直接写明文
- 条目之间**空一行**
- 参考文献章节的 `[N]` 写在行首，会被脚本打上 bookmark 不会被转超链接（已作特殊处理）
